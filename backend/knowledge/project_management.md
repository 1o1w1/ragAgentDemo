# 项目管理指南

## 项目管理概述

项目管理是将知识、技能、工具和技术应用于项目活动以满足项目要求的过程。

## 项目管理方法论

### 瀑布模型 (Waterfall)

瀑布模型是一种线性顺序的开发方法，每个阶段必须完成后才能开始下一阶段。

**阶段：**
1. 需求分析
2. 系统设计
3. 实现
4. 测试
5. 部署
6. 维护

### 敏捷开发 (Agile)

敏捷开发是一种迭代、增量的开发方法，强调灵活性和客户协作。

**核心价值观：**
- 个体和互动高于流程和工具
- 可工作的软件高于详尽的文档
- 客户协作高于合同谈判
- 响应变化高于遵循计划

### Scrum 框架

Scrum 是最流行的敏捷框架之一。

**角色：**
- **产品负责人 (Product Owner)**: 管理产品待办事项
- **Scrum Master**: 确保团队遵循 Scrum 流程
- **开发团队**: 跨职能、自组织的团队

**事件：**
- **Sprint**: 固定时间的迭代周期（通常 2-4 周）
- **Sprint 计划会议**: 规划 Sprint 工作
- **每日站会**: 同步进度和障碍
- **Sprint 评审**: 展示完成的工作
- **Sprint 回顾**: 总结改进点

### 看板方法 (Kanban)

看板方法强调可视化工作流程和限制在制品数量。

**核心实践：**
1. 可视化工作流程
2. 限制在制品数量
3. 管理流动
4. 明确流程规则
5. 实施反馈循环
6. 协作改进

## 项目规划

### 工作分解结构 (WBS)

将项目分解为可管理的工作包。

### 估算技术

- **专家判断**: 基于经验估算
- **类比估算**: 基于类似项目估算
- **三点估算**: (乐观 + 4×最可能 + 悲观) / 6
- **故事点**: 相对估算单位

### 风险管理

1. 风险识别
2. 风险分析
3. 风险应对计划
4. 风险监控

## 团队管理

### 团队发展阶段

1. **形成期**: 团队组建
2. **震荡期**: 冲突出现
3. **规范期**: 建立规范
4. **执行期**: 高效协作
5. **解散期**: 项目结束

### 沟通管理

- 定期团队会议
- 清晰的沟通渠道
- 文档化重要决策
- 及时反馈

## 项目监控

### 关键指标

- **进度**: 实际进度 vs 计划进度
- **成本**: 实际成本 vs 预算
- **质量**: 缺陷率、测试覆盖率
- **范围**: 需求变更频率

### 挣值管理 (EVM)

- **计划值 (PV)**: 计划完成工作的预算
- **挣值 (EV)**: 实际完成工作的预算
- **实际成本 (AC)**: 实际发生的成本
- **进度绩效指数 (SPI)**: EV / PV
- **成本绩效指数 (CPI)**: EV / AC

- Trello

### 协作工具

- GitHub / GitLab (代码协作)

## 最佳实践

1. 明确项目目标和范围
2. 建立清晰的沟通机制
3. 定期监控和调整
4. 管理利益相关者期望
5. 文档化重要决策和经验教训
